报错，踩坑

530 Login incorrect错误

参考：Unbuntu_FTP客户端连接响应信息：530 Login incorrect

  • 补充：
    情况一中，主要就是删掉/etc/pam.d/vsftpd 这个文件，如果使用sudo apt-get remove vsftpd时提示：
  • 提示/var/lib/dpkg/目录被占用了，可以直接到/etc/pam.d/这个目录下查看



vsftpd启动异常(code=exited, status=2)

报这个错可能导致的原因很多：

  • 配置文件配置错误/冲突，例：listen=YES和listen_ipv6=YES不能一起用，又或是你的虚拟机就没有ipv6等等。

  • 21端口已经被占用了，kill掉在使用21端口的进程

  • 配置文件格式错误，因为配置文件中有很多用#注释的语句，在其中加一行删一个#，往往很容易就导致格式错了，这时候尽管配置都正确也同样报这个错误。

其中第三种容易检查很多次配置没有错误但就找不到原因，因此一般建议用cp命令复制一份vsftpd.conf文件为vsftpd.conf_copy当作备份，然后把vsftpd.conf中注释的内容全部删除，也便于配置出错时排查错误。



WinSCP连不上ftp服务器

  • 1.先尝试能否互相ping通，若使用的是桥接模式，要在虚拟机的虚拟网络编辑器新建一个VMnetx，桥接到主机使用的网卡/网络
  • 2.连上了显示取不到目录，试试在winscp里连接时选高级，连接里把被动去掉



vi编辑器中上下左右出现ABCD错误，需要重新安装vim编辑器，顺便把剪切插件安装了

sudo apt-get install vim
//如果显示Packge ‘vim’ has no installation candidat：

//1. sudo apt-get update;

//2. sudo apt-get remove vim-common;

//3. sudo apt-get install vim

vim –version | grep clipboard //查看是否有插件


sudo apt-get install vim-gtk //安装剪切插件
